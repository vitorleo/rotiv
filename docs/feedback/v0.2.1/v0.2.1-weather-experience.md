## The Remaining Blocker: npm Packages Not Published

### What fails and why

Both `rotiv dev` and `rotiv migrate` ultimately need `@rotiv/orm` to be importable in the
user's project directory.

For `rotiv migrate`:
1. The embedded migrate-script writes a `drizzle.config.ts` pointing at `app/models/*.ts`
2. It runs `drizzle-kit generate --config drizzle.config.ts`
3. drizzle-kit uses `tsx` (via `NODE_OPTIONS=--import tsx`) to parse the model files
4. The model files contain `import { sqliteTable, text, integer } from "@rotiv/orm"`
5. `tsx` tries to resolve `@rotiv/orm` from `node_modules` — **404, package not on npm**

For `rotiv dev`:
1. The embedded route-worker's `transform.ts` SWC-compiles route files
2. It tries to resolve `@rotiv/sdk`, `@rotiv/orm`, `@rotiv/jsx-runtime` for import rewriting
3. These packages are absent — the worker cannot serve any route

### The exact error chain

```
rotiv migrate
→ spawns: node --import tsx <migrate-script> --project . --generate
→ migrate-script writes drizzle.config.ts, runs: drizzle-kit generate --config drizzle.config.ts
→ drizzle-kit parses app/models/search.ts
→ import { sqliteTable } from "@rotiv/orm"  ← ERR_MODULE_NOT_FOUND
→ drizzle-kit exits 1 with empty stderr (stdout only)
→ rotiv reports: "drizzle-kit generate failed:\n"  ← error body is empty
```

### Why this is the #1 priority

Every other tool in the framework works. Publishing 5 npm packages unblocks everything at once:
`rotiv dev`, `rotiv migrate`, `rotiv build`, and `pnpm install`. Nothing else needs to change.

## New Issue Found: Table Name Pluralization Bug

`rotiv add model Search` generated table name `searchs` instead of `searches`.

```
✓ created app/models/search.ts
  table: searchs        ← wrong, should be "searches"
```

The pluralization logic does a naive `name + "s"` without handling irregular plurals.
Common cases that will break: `search→searchs`, `category→categorys`, `query→querys`,
`status→statuss`, `match→matchs`.

**Suggested fix:** Use a proper English pluralization library (e.g. `pluralize`) or a small
lookup table for the most common irregular cases.

## New Issue: `rotiv migrate` Error Body Is Still Empty

Despite the v0.2.1 fix note saying "capture both stdout and stderr", the error message is still:
```
error [E_UNKNOWN] migration error: Migration failed: {"error":"Error: drizzle-kit generate failed:\n"}
```
The `\n` at the end with no content means the error is still being lost. In this specific case,
drizzle-kit crashes during `tsx` ESM resolution and writes the error to the Node.js uncaught
exception handler, which may bypass both stdout and stderr of the drizzle-kit process itself.
The `spawnSync` result has empty `stderr` and `stdout` because the error happens in the
`tsx` loader before drizzle-kit even starts.
