## New Blocker: Windows Path Not Converted to `file://` URL

**Error (both `rotiv dev` and `rotiv migrate`):**
```
Error [ERR_UNSUPPORTED_ESM_URL_SCHEME]: Only URLs with a scheme in: file, data, and node are
supported by the default ESM loader. On Windows, absolute paths must be valid file:// URLs.
Received protocol 'c:'
```

### Root cause

When `resolve_tsx_loader()` finds tsx at e.g.:
```
C:\Users\Vitor\AppData\Roaming\npm\node_modules\tsx\dist\esm\index.cjs
```

It passes this raw Windows path to Node.js as:
```
node --import C:\Users\Vitor\AppData\Roaming\npm\node_modules\tsx\dist\esm\index.cjs
```

Node.js ESM loader interprets the path as a URL. On Windows, `C:\...` looks like a URL with
scheme `c:`, which is not `file:`, `data:`, or `node:` — so it throws.

### Required fix (one function)

The path must be converted to a `file://` URL before being passed to `--import`:

```rust
fn path_to_file_url(path: &Path) -> String {
    // On Windows: C:\foo\bar → file:///C:/foo/bar
    // On Unix:    /foo/bar  → file:///foo/bar
    let normalized = path.to_string_lossy().replace('\\', "/");
    if normalized.starts_with('/') {
        format!("file://{}", normalized)
    } else {
        // Windows drive letter: C:/foo/bar → file:///C:/foo/bar
        format!("file:///{}", normalized)
    }
}
```

This is a one-function fix in `worker.rs` and `migration.rs` wherever the resolved tsx path
is passed to the `node --import` argument.

Note: Rust's `std::path::Path` has `to_string_lossy()` but not a direct `to_file_url()` method.
The `url` crate's `Url::from_file_path()` handles this correctly cross-platform.

## One Additional Note: `better-sqlite3` Native Build

After `pnpm install`, `better-sqlite3` needs its native addon compiled. pnpm v10 requires
explicit build approval:

```
pnpm approve-builds  # interactive prompt to allow better-sqlite3, esbuild
```

This is a pnpm security feature (not a Rotiv bug), but agents can't use interactive prompts.
The workaround I used was adding to `.npmrc`:
```
onlyBuiltDependencies[]=better-sqlite3
onlyBuiltDependencies[]=esbuild
```

Consider adding these lines to the `.npmrc` written by `rotiv new`, or documenting this step.
