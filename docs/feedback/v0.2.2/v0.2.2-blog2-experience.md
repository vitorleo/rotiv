## TL;DR

The app code is complete and correct. `rotiv validate` passes. But neither `rotiv dev` nor
`rotiv migrate` can run because `tsx` is not resolvable — which is a direct consequence of
`@rotiv/*` npm packages not being published. **This is the same root blocker present in every
release since v0.1.0.**

---

## What Happened

### Scaffolding (✓ perfect)

The v0.2.2 pluralization fix confirmed working: `Post → posts` ✓

### Running the app (✗ blocked)

**`rotiv migrate`:**
```
error: Cannot find package 'tsx' imported from …\blog2\
```

**`rotiv dev`:**
```
error: Cannot find package 'tsx' imported from …\.tmps9LQZ4\
```

Both commands spawn Node.js with `--import tsx`. Node resolves `tsx` from the project's
`node_modules`, which is empty because `pnpm install` cannot run (all `@rotiv/*` deps are 404
on npm).

---

## Root Cause (Unchanged Since v0.1.0)

```
pnpm install
  → GET @rotiv/sdk        404 Not Found
  → GET @rotiv/orm        404 Not Found
  → GET @rotiv/jsx-runtime 404 Not Found
  → GET @rotiv/signals    404 Not Found
  → GET @rotiv/types      404 Not Found
  → INSTALL FAILS

rotiv dev / rotiv migrate
  → node --import tsx ...
  → Cannot find package 'tsx'  (not in node_modules because install failed)
  → CRASH
```

The framework is fully functional from the CLI's perspective. The single missing piece is
publishing the 5 `@rotiv/*` npm packages. Everything else is working correctly.

## The One Fix Needed

Publish these 5 packages to npm (or a GitHub Package Registry with documented `.npmrc`):

- `@rotiv/sdk`
- `@rotiv/orm`
- `@rotiv/jsx-runtime`
- `@rotiv/signals`
- `@rotiv/types`

The v0.2.0 changelog stated this was done but the npm registry still returns 404 for all of them.
The v0.2.2 changelog notes that the `NPM_TOKEN` secret needs to be set in GitHub Actions to
trigger the publish workflow — this appears to still not have been done.

## How to Run (once npm packages are published)

```bash
cd blog2
pnpm install        # will work once @rotiv/* are on npm
rotiv migrate       # creates app/.rotiv/dev.db with posts table
rotiv dev           # starts dev server at http://localhost:3000
```

Then open http://localhost:3000:
- Click **+ New post** to create a post
- Fill in title, author, and body — press **Publish**
- You're redirected to the post's page at `/posts/<slug>`
- Click **← Back to blog** to see the full post list
