### ✗ `rotiv migrate` fails — `better-sqlite3` native bindings not built

**Actual error:**
```
Error: Could not locate the bindings file.
 → ...better-sqlite3\build\Release\better_sqlite3.node
 → ...better-sqlite3\lib\binding\node-v127-win32-x64\better_sqlite3.node
 (13 paths tried, all missing)
```

**Root cause:** `better-sqlite3` is a native Node.js addon that must be compiled from C++ for the
current Node.js version and platform. pnpm v10.32.1 skipped the build step despite the `.npmrc`:

```ini
onlyBuiltDependencies[]=better-sqlite3
onlyBuiltDependencies[]=esbuild
```

pnpm v10 still outputs: `"Ignored build scripts: better-sqlite3@9.6.0 ... Run pnpm approve-builds"`

The `onlyBuiltDependencies[]` array syntax in `.npmrc` appears not to be the correct pnpm v10
configuration key. The correct key for pnpm v10 is `onlyBuiltDependencies` as a JSON-like list in
`package.json` under `pnpm.onlyBuiltDependencies`, not in `.npmrc`.

**Fix options:**
1. Add to `package.json`:
   ```json
   "pnpm": {
     "onlyBuiltDependencies": ["better-sqlite3", "esbuild"]
   }
   ```
2. Remove `onlyBuiltDependencies[]` from `.npmrc` (it has no effect and is misleading)
3. Or instruct users to run `pnpm approve-builds` after `pnpm install`

## `better-sqlite3` Native Build — pnpm v10 Config Key

The correct way to pre-approve native builds in pnpm v10 is via `package.json`, not `.npmrc`:

```json
{
  "pnpm": {
    "onlyBuiltDependencies": ["better-sqlite3", "esbuild"]
  }
}
```

Reference: https://pnpm.io/package_json#pnmmonlybuiltdependencies
