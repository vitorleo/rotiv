# v0.2.1 Changelog

Released: 2026-03-15

## Summary

Patch release fixing three bugs found during v0.2.0 AI agent testing (blog app experience report).

---

## Bug Fixes

### Bug 1: `rotiv dev` crashes with "Top-level await not supported with CJS format"

The embedded route-worker was extracted to a temp directory without a `package.json`, causing
Node.js to treat it as CommonJS. The worker uses top-level `await`, which is ESM-only.

**Fix:** Write `{"type":"module"}` to the temp directory root after extracting worker files.

Same fix applied to `rotiv migrate`'s embedded migrate-script extraction.

Files: `crates/rotiv-cli/src/commands/dev.rs`, `crates/rotiv-cli/src/commands/migrate.rs`

### Bug 3: `rotiv migrate` error message is blank when drizzle-kit writes to stdout

When `drizzle-kit` failed, the error reported was `"Migration failed:\n"` with no useful detail.
Root cause: the runner only captured stderr, but drizzle-kit writes some errors to stdout.

**Fix:** Capture both stdout and stderr from the child process; include whichever is non-empty
(both if both have content). Falls back to `"no output from drizzle-kit"` if both are empty.

File: `crates/rotiv-orm/src/migration.rs`
