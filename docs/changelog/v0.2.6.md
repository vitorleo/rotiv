# Rotiv v0.2.6 — Changelog

**Released:** 2026-03-16

## Summary

v0.2.6 fixes `rotiv build` failing with "Build script not found" in standalone projects, and
correctly configures pnpm v10 to auto-compile `better-sqlite3` native bindings.

---

## Fixes

### 1. `rotiv build` now works in standalone projects — FIXED

`rotiv build` previously failed with:
```
error [E_UNKNOWN] build failed: Build script not found.
Set ROTIV_BUILD_SCRIPT_PATH or run from the Rotiv monorepo
```

The build-script was only resolved from the monorepo dev layout or via env var — there was no
production fallback and no embedded copy in the binary (unlike the worker and migrate-script
which were already embedded in v0.2.0).

**Fix:** `build.rs` now embeds the three build-script TypeScript source files at compile time
(`index.ts`, `compiler.ts`, `manifest.ts`) using `include_str!()`, writes them to a temp
directory at runtime, and passes the entry path to `compile_project()`. The same
`resolve_tsx_loader()` + `path_to_file_url_or_bare()` treatment is applied so the build
works cross-platform including on Windows with globally installed tsx.

`CompileOptions` gains two new fields:
- `script_path: Option<PathBuf>` — explicit script path (takes priority over auto-resolution)
- `tsx_loader: String` — the resolved tsx specifier (file:// URL or bare name)

### 2. `better-sqlite3` native bindings now compiled by pnpm v10 — FIXED

In v0.2.5, `.npmrc` used `onlyBuiltDependencies[]=better-sqlite3` which has no effect in
pnpm v10. pnpm v10 reads this setting from `package.json` under the `"pnpm"` key:

```json
"pnpm": {
  "onlyBuiltDependencies": ["better-sqlite3", "esbuild"]
}
```

**Fix:** The scaffold `package.json` now includes this `"pnpm"` configuration block.
The ineffective `onlyBuiltDependencies[]` lines have been removed from `.npmrc`.

Without this fix, `rotiv migrate` and `rotiv dev` fail with:
```
Error: Could not locate the bindings file.
better-sqlite3\build\Release\better_sqlite3.node (13 paths tried, all missing)
```
