# Rotiv v0.2.5 — Changelog

**Released:** 2026-03-16

## Summary

v0.2.5 fixes a Windows-specific crash that blocked `rotiv dev` and `rotiv migrate` after the
tsx resolver was fixed in v0.2.4. It also makes `pnpm install` fully non-interactive on
pnpm v10 by pre-approving native builds.

---

## Fixes

### 1. `ERR_UNSUPPORTED_ESM_URL_SCHEME` on Windows — FIXED

When `resolve_tsx_loader()` found tsx via `npm root -g` or `which tsx`, it returned a raw
Windows path (e.g. `C:\Users\Vitor\AppData\Roaming\npm\node_modules\tsx\dist\esm\index.cjs`).

Node.js ESM loader parses the `--import` argument as a URL. On Windows, `C:\...` looks like
a URL with scheme `c:`, which is not an allowed scheme (`file:`, `data:`, `node:`):

```
Error [ERR_UNSUPPORTED_ESM_URL_SCHEME]: Only URLs with a scheme in: file, data, and node are
supported by the default ESM loader. On Windows, absolute paths must be valid file:// URLs.
Received protocol 'c:'
```

**Fix:** A new `path_to_file_url_or_bare()` helper converts absolute paths to `file://` URLs
before they are passed to `node --import`:

- `C:\Users\...\index.cjs` → `file:///C:/Users/.../index.cjs`
- `/usr/local/.../index.cjs` → `file:///usr/local/.../index.cjs`
- Bare names like `"tsx"` are returned unchanged (Node resolves those via package resolution)

Applied in both `rotiv-core::worker` and `rotiv-orm::migration`.

### 2. pnpm v10 native build approval — FIXED

pnpm v10 introduced interactive approval for packages that run build scripts (native addons).
Without pre-approval, `pnpm install` would pause waiting for user input — unusable in
automated/agent contexts.

The `.npmrc` written by `rotiv new` now includes:

```
onlyBuiltDependencies[]=better-sqlite3
onlyBuiltDependencies[]=esbuild
```

This tells pnpm v10 to auto-approve native builds for these two packages, making
`pnpm install` fully non-interactive.
