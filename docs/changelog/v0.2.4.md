# Rotiv v0.2.4 — Changelog

**Released:** 2026-03-15

## Summary

v0.2.4 fixes the two remaining blockers that prevented `rotiv dev` and `rotiv migrate` from
working when `@rotiv/*` packages are not yet published to npm.

---

## Fixes

### 1. `pnpm install` now succeeds without `@rotiv/*` on npm

The scaffold `package.json` now puts `@rotiv/*` packages under `optionalDependencies` instead
of `dependencies`. npm/pnpm skip optional packages that fail to resolve, so `pnpm install`
completes successfully and installs the real packages (`tsx`, `drizzle-kit`, `drizzle-orm`,
`better-sqlite3`) even when `@rotiv/*` is not on the registry.

A `.npmrc` file is also written to every new project with `optional=true` to reinforce this
behaviour across all package managers.

**Before:** `pnpm install` failed with `404 Not Found` for every `@rotiv/*` package, leaving
`node_modules` empty (no `tsx`, no `drizzle-kit`).

**After:** `pnpm install` installs the non-`@rotiv/*` packages successfully; `tsx` and
`drizzle-kit` are available in `node_modules`.

### 2. `resolve_tsx_loader()` — global npm path now searched

On Windows (and other systems where `tsx` is installed globally via `npm install -g tsx`),
the global npm package directory is `%APPDATA%\Roaming\npm\node_modules\`. This was not
searched in v0.2.3, causing `node --import tsx` to fail with:

```
Error [ERR_MODULE_NOT_FOUND]: Cannot find package 'tsx'
```

v0.2.4 adds two additional resolution steps in both `rotiv-core::worker` and
`rotiv-orm::migration`:

3. **`npm root -g`** — asks npm for its global package root at runtime; checks
   `<npm_root>/tsx/dist/esm/index.cjs`
4. **`where tsx` / `which tsx`** — locates the tsx binary on PATH and walks up the directory
   tree to find `node_modules/tsx/dist/esm/index.cjs`

The bare `"tsx"` string is kept as a last-resort fallback (works only when tsx is in the
project's own `node_modules`).

Full resolution order (unchanged from v0.2.3 for steps 1–2):
1. `<project>/node_modules/tsx/dist/esm/index.cjs`
2. `<binary_dir>/node_modules/tsx/dist/esm/index.cjs`
3. `<binary_dir>/../node_modules/tsx/dist/esm/index.cjs`
4. **NEW** `npm root -g` → `<npm_global>/tsx/dist/esm/index.cjs`
5. **NEW** `which tsx` / `where tsx` → walk up to find package root
6. Bare `"tsx"` fallback

---

## Other Changes

- `rotiv new` now writes `.npmrc` to the project root (embedded via `include_str!`)
- `rotiv new` next-steps message now shows `rotiv migrate` before `rotiv dev`

---

## Upgrade Notes

No breaking changes. Existing projects can add `.npmrc` manually:

```
# @rotiv/* packages are optional — install will succeed even if they are not yet on npm.
optional=true
```

And move `@rotiv/*` lines from `dependencies` to `optionalDependencies` in `package.json`.
