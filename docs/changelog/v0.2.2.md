## Bug Fixes

### Bug 1: `rotiv add model Search` generates table name `searchs` instead of `searches`

The pluralization logic used a naive rule that only handled `s/x/z → es` and defaulted to `+s`.
This produced wrong table names for many common model names.

**Broken:** `Search→searchs`, `Category→categorys`, `Query→querys`, `Match→matchs`, `Status→statuss`

**Fixed:**
- `ch/sh` endings → `es`: `Search→searches`, `Match→matches`
- consonant+`y` endings → `ies`: `Category→categories`, `Query→queries`, `Entry→entries`
- `s/x/z` endings → `es`: `Status→statuses`, `Box→boxes`
- Default: `+s` (unchanged)

File: `crates/rotiv-cli/src/commands/add.rs`

### Bug 2: `rotiv migrate` error message still blank when tsx/drizzle-kit fail

Despite the v0.2.1 fix, error messages were still empty in one case: when Node.js fails to
resolve a module (e.g. `@rotiv/orm` not installed), the crash happens before drizzle-kit runs.
Node sets `result.error` on the `spawnSync` result and leaves both `stderr` and `stdout` empty.
The previous code used `result.stderr ?? result.stdout` which fails when `stderr` is `""`.

**Fixed:**
- Check `result.error` first and surface its message directly
- Use `[stderr, stdout].filter(Boolean).join("\n")` instead of `??` to pick up whichever stream has content

File: `packages/@rotiv/migrate-script/src/runner.ts`
