# v0.2.0 Changelog

Released: 2026-03-15

## Summary

v0.2.0 is the first standalone-ready release of Rotiv. Previously, `rotiv dev` and `rotiv migrate` required the monorepo to be present on disk because they resolved TypeScript worker and migration scripts relative to the binary's install path. v0.2.0 fixes this by embedding all required TypeScript source directly into the binary, making it fully self-contained.

npm packages for `@rotiv/types`, `@rotiv/sdk`, `@rotiv/jsx-runtime`, `@rotiv/signals`, and `@rotiv/orm` are now properly built and published.

---

## Breaking Changes

None. The CLI interface is unchanged.

---

## Bug Fixes

### Blocker 1: `rotiv dev` fails outside monorepo (E001)

**Root cause:** `RouteWorker` resolved the route worker entry point by guessing a path relative to the binary — `../../packages/@rotiv/route-worker/src/index.ts`. This path doesn't exist outside the development monorepo.

**Fix:** Embed all 6 route-worker TypeScript source files into the binary at compile time via `include_str!()`. At runtime, `rotiv dev` writes them to a temporary directory and passes the explicit path to the worker resolver. The resolver now accepts an `embedded_path: Option<PathBuf>` parameter and prefers it over all guessed paths.

Files changed:
- `crates/rotiv-cli/src/commands/dev.rs` — embeds worker source, writes to tempdir
- `crates/rotiv-core/src/worker.rs` — `RouteWorker::new()` + `resolve_worker_path()` updated
- `crates/rotiv-core/src/server.rs` — `DevServerConfig.worker_path` field added
- `crates/rotiv-cli/Cargo.toml` — `tempfile = "3"` moved to `[dependencies]`

### Blocker 2: npm packages not on npm registry

**Root cause:** All `packages/@rotiv/*` `package.json` files pointed `exports` at `./src/index.ts` and had no `build` or `prepublishOnly` scripts. The packages were never published.

**Fix:** Updated all five public packages to point `exports` to `./dist/index.js`, added `"files": ["dist"]`, `"build": "tsc --project tsconfig.build.json"`, and `"prepublishOnly": "pnpm build"`. Created `tsconfig.build.json` for packages that lacked one. Added `.github/workflows/npm-publish.yml` to publish all packages on GitHub release.

Packages updated: `@rotiv/types`, `@rotiv/sdk`, `@rotiv/jsx-runtime`, `@rotiv/signals`, `@rotiv/orm`

### Blocker 3: `rotiv migrate` fails outside monorepo (E015)

**Root cause:** Same pattern as Blocker 1 — migration script path was guessed relative to the binary.

**Fix:** Embed 3 migrate-script TypeScript source files via `include_str!()` in `migrate.rs`. Write to tempdir at runtime, pass explicit path via `MigrationOptions.script_path`.

Files changed:
- `crates/rotiv-cli/src/commands/migrate.rs` — embeds migrate source, writes to tempdir
- `crates/rotiv-orm/src/migration.rs` — `MigrationOptions.script_path` field + `resolve_migrate_script_path()` updated

## Minor Improvements

### Dev banner shows relative file paths

The `rotiv dev` startup banner previously showed only the filename (e.g. `[id].tsx`) for each route. It now shows the full relative path from the project root (e.g. `app/routes/users/[id].tsx`).

Files changed: `crates/rotiv-core/src/server.rs`

### E011/E012 errors now include corrected suggestions

`rotiv add model <name>` (E011) and `rotiv add module <name>` (E012) now compute a suggested corrected name and include it in both the `suggestion` and `corrected_code` fields of the error.

- E011: `user_profile` → suggests `UserProfile`
- E012: `MyModule` → suggests `my-module`

Files changed:
- `crates/rotiv-core/src/error.rs` — added `with_corrected_code()` builder
- `crates/rotiv-cli/src/commands/add.rs` — added `to_pascal_case()`, `to_kebab_case()` helpers; updated E011/E012

## Infrastructure

### Release workflow builds npm packages before Rust binary

The GitHub Actions release workflow (`release.yml`) now runs `pnpm install` and `pnpm -r build` to build TypeScript packages before compiling the Rust binary. This ensures the embedded `include_str!()` paths resolve correctly in CI.

### New project template uses `^0.2.0`

The `rotiv new` scaffolded `package.json` now references `^0.2.0` for all `@rotiv/*` dependencies instead of `^0.1.0`.
